    • Billing/Invoice Management

Manage customer invoices and process bill payments using 3rd party payment platforms.

  • Data Collection: Call Data Records and Licenses are gathered from platform and provisioning service for each customer. These are Kafka based messages which are processed and stored in our databases.

  • Usage Calculation: This data is used to generate and feed usage data.

  • Invoicing:
    • Payment Platform Integration: The system connects with a payment platform to fetch relevant invoice details.
    • Invoice Generation: Invoices are generated based on the services used and calculated usage.
  • Amount Due Check: The system assesses whether there is an amount due.
    • No Amount Due: If the amount is zero, the process ends.
    • Amount Due:
      • Messaging: A notification is sent to a messaging service (Kafka).
      • Billing Service: The message is consumed by a billing service.
      • Payment Data Retrieval: The system connects again with the payment platform to get invoice data.
      • ID Retrieval: Invoice IDs are obtained and will be used as primary keys.
  • Data Storage & Extraction:
    • Database Storage: Invoice data is stored in a database.
    • REST API Usage: REST APIs are employed to extract specific data using filters.
    • AWS S3 Storage: Data is also stored in AWS S3 buckets for potential future use.
  • Customer Interaction:
    • Frontend Feeding: Data is fed to a frontend billing application.
    • Customer Actions: Customers can either view invoices or use an auto-pay feature.
  • Payment Processing:
    • Payment Platform Notification: The payment platform is notified.
    • Payment Status Update: The payment status is updated in the system.
    • Transaction Successful: The process ends successfully.
    • Transaction Unsuccessful:
      • Customer Emails: Emails are sent to customers using SendGrid.
      • Abort and Notify: The process is aborted, the payment platform is notified, and a PagerDuty alert is raised.
